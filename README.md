## 🎯 系统概述

Alpha Hive 是一个**去中心化、蜂群智能驱动**的投资研究系统，通过并行的 AI Agent 集合自动扫描市场机会。

### 核心特点

✅ **多源数据融合**
- SEC 交易披露（Form 4 / 13F）
- Polymarket 预测市场赔率
- X（推特）舆情与情绪分析
- 财报、事件催化剂、产业竞争格局

✅ **蜂群智能架构**
- 6 种专业 Agent 角色协作
- 信息素机制（高价值信号自动传播）
- 涌现式决策（无中央决策者）

✅ **结构化输出**
- 8 部分标准化简报
- 综合评分（0~10 分）+ 置信度
- 风险提示与失效条件明确

✅ **实时更新**
- 每日凌晨自动扫描
- GitHub Pages 自动部署
- 免费托管，无需服务器

## 📄 简报结构（8 部分）

每份深度简报都遵循标准化的 8 部分结构，确保信息完整、可比较。

### 1. 聪慧之力（Smart Money Signals）
🔹 **内容**: 机构投资人、对冲基金最新持仓变化
🔹 **来源**: SEC Form 4 / 13F 披露
🔹 **关键看点**:
- 知名投资人是否新增持仓？
- 持仓增减趋势如何？
- 增仓比例有多大？

**示例** (NVDA): 巴菲特旗下伯克希尔哈撒韦增持 23% → 机构看好信号

---

### 2. 市场预期（Market Expectations）
🔹 **内容**: 分析师一致预期、Polymarket 赔率
🔹 **来源**: 彭博、Polymarket、投资平台
🔹 **关键看点**:
- 分析师目标价 vs 当前价
- 共识方向是否一致？
- 市场隐含的成功概率？

**示例** (VKTX): 50+ 分析师 83% 给出"买入"评级，暗示市场有定价缺陷

---

### 3. X 舆论温度（X Sentiment Analysis）
🔹 **内容**: 社交媒体讨论热度、情绪方向
🔹 **来源**: X（推特）公开数据
🔹 **关键看点**:
- 讨论量是否上升（关注度）？
- 正面评论占比（情绪）？
- KOL 观点是否一致？

**示例** (TSLA): 每日提及量 8,000+，但正面评论占比仅 45% → 存在疑虑

---

### 4. 催化剂日期（Catalysts & Timelines）
🔹 **内容**: 未来 1~8 周的关键事件
🔹 **来源**: 公司公告、行业日历、监管预告
🔹 **关键看点**:
- 哪些事件可能触发价格波动？
- 催化剂时间窗有多清晰？
- 事件结果的上/下行空间？

**示例** (NVDA): 财报发布（2 周内）→ 涨幅预期 8~12% 或下跌 5~8%

---

### 5. 竞争态势（Competitive Landscape）
🔹 **内容**: 行业格局、竞争对手动向
🔹 **来源**: 行业报告、竞品财报、专利/产品发布
🔹 **关键看点**:
- 标的的市场份额是否稳定？
- 新竞争对手出现？
- 产业链是否有风险？

**示例** (NVDA): 市占率 80% 但面临 AMD、英特尔竞争加剧 → 中期风险

---

### 6. 综合判断（Synthesis）
🔹 **内容**: 多 Agent 投票结果、共识度、分歧观点
🔹 **来源**: 6 个 Agent 的独立评分与权重
🔹 **关键看点**:
- 6 个 Agent 中有几个看多？
- 是否存在关键分歧？
- 少数派意见是什么？

**示例** (NVDA): 6/6 Agent 同向看多（100% 一致性）→ 高置信度

---

### 7. 推荐行动（Recommendations）
🔹 **内容**: 具体行动建议与风险上限
🔹 **来源**: 综合分析输出
🔹 **关键看点**:
- 建议的方向与力度？
- 最大回撤风险有多大？
- 仓位上限建议？

**示例**: "建议 Long，目标价 $650，止损 $580，单仓不超过 3%"

---

### 8. 免责与风险（Disclaimers & Risk Management）
🔹 **内容**: 法律免责、假设前提、失效条件
🔹 **来源**: 合规标准化文本
🔹 **关键看点**:
- 本分析不构成投资建议
- 关键假设是什么？
- 哪些情况下分析会失效？

## 📈 评分算法

每份报告的综合分数（0~10）采用以下公式计算：

```
综合评分 = 0.30×信号强度 + 0.20×催化剂清晰度 + 0.20×舆情热度
          + 0.15×赔率错配 + 0.15×风险调整
```

### 权重解释

| 维度 | 权重 | 说明 |
|------|------|------|
| **信号强度** | 30% | 披露 + 基本面的共振程度 |
| **催化剂清晰度** | 20% | 未来事件触发概率与时间确定性 |
| **舆情热度** | 20% | X 讨论热度 + 情绪方向 |
| **赔率错配** | 15% | Polymarket 隐含概率 vs 基本面 |
| **风险调整** | 15% | 波动率、流动性、拥挤度反向调整 |

### 评分区间

| 分数段 | 等级 | 建议 |
|--------|------|------|
| 8.5~10.0 | ⭐⭐⭐⭐⭐ 极强信号 | 优先跟踪，考虑持仓 |
| 7.5~8.4 | ⭐⭐⭐⭐ 强信号 | 值得关注，等待切入机会 |
| 6.5~7.4 | ⭐⭐⭐ 中等信号 | 观察，补充证据 |
| 5.0~6.4 | ⭐⭐ 弱信号 | 证据不足，不行动 |
| < 5.0 | ⭐ 极弱信号 | 反向操作或规避 |

## 🐝 蜂群架构（6 Agent 角色）

Alpha Hive 由 6 个专业 Agent 角色协作，每个 Agent 独立采集和分析，最后通过民主投票和加权合成生成最终评分。

### Agent 角色

| 角色 | 任务 | 数据来源 | 独立评分 |
|------|------|---------|---------|
| **Scout 侦察蜂** | SEC 披露数据采集与核实 | Form 4 / 13F | Signal 维度 |
| **SentimentBee 情绪蜂** | X 平台舆论分析与量化 | X API / Web Scraping | Sentiment 维度 |
| **OddsBee 赔率蜂** | Polymarket 市场概率解读 | Polymarket API | Odds 维度 |
| **CatalystBee 催化蜂** | 事件时间窗梳理与评估 | 公司公告 / 日历 | Catalyst 维度 |
| **CrossBee 交叉蜂** | 多源信号共振矩阵构建 | 综合多源 | 一致性验证 |
| **ValidatorBee 校验蜂** | 事实核实与风险评估 | 所有来源 | 置信度调整 |

### 协作流程

```
┌─────────────────────────────────────────────────┐
│ Phase 1: 任务分解 (Task Decomposition)          │
│ → 明确目标标的、事件、时间窗                      │
└────────────────┬────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────────┐
│ Phase 2: 并行觅食 (Parallel Foraging)           │
│ → 6 个 Agent 同步采集不同数据源                   │
│   Scout → SEC | SentimentBee → X | OddsBee → Polymarket
│   CatalystBee → 事件 | CrossBee → 共振 | ValidatorBee → 校验
└────────────────┬────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────────┐
│ Phase 3: 交叉共振 (Signal Resonance)            │
│ → 识别跨源一致性与冲突点                         │
│ → CrossBee 建立共振矩阵                         │
└────────────────┬────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────────┐
│ Phase 4: 校验与投票 (Validation & Voting)       │
│ → ValidatorBee 核对事实                         │
│ → 6 个 Agent 各自评分 (0~10)                    │
└────────────────┬────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────────┐
│ Phase 5: 蒸馏与合成 (Distillation)              │
│ → 多数投票 + 加权合成                           │
│ → 生成最终报告                                  │
└────────────────┬────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────────┐
│ Phase 6: 反馈进化 (Feedback Loop)               │
│ → T+1、T+7、T+30 回溯预测偏差                  │
│ → 优化权重，提升后续准确率                       │
└─────────────────────────────────────────────────┘
```

## 🛡️ 风险与免责

⚠️ **重要提示**

本系统生成的所有分析**不构成投资建议**，仅为公开信息研究与情景推演。

### 使用须知

1. **非投资顾问服务**
   - 无法替代持牌金融顾问
   - 不能用于正式投资决策唯一依据

2. **数据风险**
   - SEC 披露可能存在延迟
   - X 舆论可能包含噪音与虚假信息
   - Polymarket 赔率非实时更新

3. **市场风险**
   - 股票价格波动剧烈，存在本金损失风险
   - 催化剂可能延迟发生或不发生
   - 竞争格局与监管环境随时变化

4. **仓位建议**
   - 建议单仓不超过 3%~5% 账户价值
   - 高评分标的仍需止损保护
   - 多仓位分散风险
